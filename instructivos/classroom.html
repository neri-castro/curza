¿Qué es GitHub Classroom?

GitHub Classroom es una herramienta gratuita que potencia **GitHub** para el entorno educativo. Su misión es simplificar drásticamente la gestión de tareas, la distribución de código y la retroalimentación entre profesores y estudiantes. Olvídate de los correos y las plataformas complejas; con Classroom, cada estudiante o grupo recibe su propio repositorio de código, permitiendo un seguimiento claro del progreso y un control de versiones impecable.

Los profesores pueden crear plantillas para cada tarea, asegurando que todos comiencen con la misma base. A partir de ahí, pueden observar el progreso en tiempo real, revisar el código y ofrecer feedback directamente donde importa. Esta aplicación interactiva te guiará a través de todo lo que necesitas saber para empezar.

Guía de Configuración para Profesores

Sigue estos pasos para tener tu clase lista y funcionando en minutos.

Paso 1: Acceso y Autorización

▼

Primero, dirígete a la página de **GitHub Classroom** e inicia sesión con tu cuenta de GitHub. Deberás autorizar a la aplicación para que acceda a tu cuenta, lo que le permite gestionar repositorios y organizaciones en tu nombre.

Paso 2: Crear una Organización

▼

GitHub Classroom necesita una **Organización** para agrupar todos los repositorios de tu clase. Esto mantiene el trabajo de tus estudiantes separado de tus proyectos personales. Puedes crear una nueva organización directamente desde Classroom si aún no tienes una.

Paso 3: Crear tu "Classroom"

▼

Un "Classroom" es el espacio de trabajo para un curso específico. Al crearlo, le darás un nombre (ej: "Desarrollo Web 2024") y lo asociarás a la organización que creaste en el paso anterior. Opcionalmente, puedes vincularlo a tu LMS (como Moodle o Canvas) para importar listas de estudiantes.

Paso 4: Crear una Asignación

▼

Este es el corazón de la herramienta. Una asignación es una tarea. Deberás definir:

  • **Nombre:** "Tarea 1: Portfolio Personal".
  • **Tipo:** Individual o Grupal.
  • **Repositorio base:** Puedes empezar con un repositorio vacío o usar una plantilla con código de inicio, instrucciones y archivos de configuración.
  • **Fecha límite:** Opcional, pero recomendado.

Paso 5: Distribuir y Monitorear

▼

Classroom genera un enlace de invitación único para cada asignación. Comparte este enlace con tus estudiantes. A medida que aceptan la tarea, podrás ver cómo se crean sus repositorios y seguir su progreso directamente desde el panel de la asignación.

Autocorrección con GitHub Actions

Automatiza la revisión de tareas y el feedback instantáneo para tus estudiantes.

Una de las funcionalidades más potentes de GitHub Classroom es la **autocorrección**. Esto se logra integrando **GitHub Actions**, un servicio de automatización que permite ejecutar flujos de trabajo (workflows) en tu repositorio. Cuando un estudiante sube su código (`git push`), una GitHub Action puede ejecutar scripts de prueba, verificar el estilo del código o incluso compilarlo, dando un feedback inmediato y automático.

Para habilitar la autocorrección, debes añadir un archivo de configuración en tu repositorio de plantilla, dentro de la carpeta `.github/workflows/`. Este archivo tiene un formato **YAML** y define los pasos que se ejecutarán. A continuación, se muestra un ejemplo simple que comprueba si un archivo específico existe y que el código pasa un linter. 

name: Autocorrección de Tarea
on: [push]
jobs:
  check-files:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v3

      - name: Check for 'index.html' file
        run: |
          if [ ! -f "index.html" ]; then
            echo "Error: El archivo 'index.html' no se encontró."
            exit 1
          fi

      - name: Check for 'style.css' file
        run: |
          if [ ! -f "style.css" ]; then
            echo "Error: El archivo 'style.css' no se encontró."
            exit 1
          fi
          
      - name: Run a simple linting check
        run: |
          echo "Ejecutando un linter básico..."
          # Aquí podrías instalar y ejecutar un linter real,
          # por ejemplo, para JavaScript o Python.
          # Ejemplo: npx eslint .
          echo "Linter completado con éxito."

Una vez que este archivo se incluye en tu repositorio de plantilla, cada vez que un estudiante haga un `push`, GitHub ejecutará automáticamente este flujo de trabajo. Los resultados (éxito o error) serán visibles en la pestaña de "Actions" del repositorio del estudiante, brindándoles una retroalimentación instantánea sobre su trabajo.

Integración con GitHub Codespaces

Elimina la configuración local y permite a los estudiantes empezar a codificar al instante.

**GitHub Codespaces** es un entorno de desarrollo en la nube que se ejecuta directamente desde tu navegador. Al integrarlo con GitHub Classroom, eliminas el mayor obstáculo para los estudiantes: la configuración de sus entornos locales. No más problemas de versiones, dependencias o sistemas operativos diferentes.

Configuración por el Profesor

Cuando creas una nueva asignación en GitHub Classroom, tienes la opción de "Habilitar un entorno de desarrollo integrado (IDE) en la nube". Al seleccionar esta opción y elegir **Codespaces**, puedes definir un archivo de configuración (`devcontainer.json`) en tu repositorio de plantilla que especifica las extensiones, librerías y configuraciones necesarias para la tarea.

Experiencia del Estudiante

Cuando un estudiante acepta la asignación con Codespaces habilitado, verá un botón para abrir su entorno de desarrollo. Con un solo clic, se lanza un entorno completo en el navegador que ya contiene el código base, todas las dependencias instaladas y las herramientas listas para usar. El estudiante puede empezar a trabajar de inmediato, sin necesidad de instalar nada en su máquina local.

Ejemplo Práctico: Sitio Web Personal

Observa el flujo de trabajo desde la perspectiva del profesor y del estudiante.

Rol del Profesor 🧑‍🏫

1. Prepara la plantilla: Crea un repositorio en GitHub (`plantilla-web-personal`) con un `index.html`, `style.css`, un archivo de autocorrección y un archivo de configuración para Codespaces (`devcontainer.json`).

2. Crea la asignación: En Classroom, crea la tarea "Sitio Web Personal", la configura como individual y selecciona el repositorio plantilla, habilitando la opción de Codespaces.

3. Comparte el enlace: Copia el enlace de invitación y lo publica en el canal de la clase.

4. Monitorea el progreso y la autocorrección: Desde el panel de Classroom, ve los repositorios de los estudiantes y revisa los resultados de las Actions para un feedback rápido.

5. Da feedback personalizado: Abre "Pull Requests" o deja comentarios en líneas de código específicas para guiar al estudiante, complementando la autocorrección.

Rol del Estudiante 🧑‍🎓

1. Acepta la tarea: Hace clic en el enlace de invitación y acepta la asignación en la página de GitHub Classroom.

2. Recibe su repositorio: Automáticamente, se crea su repositorio personal (`sitio-web-personal-username`), incluyendo la plantilla y los archivos de autocorrección.

3. Trabaja en el código: Abre el Codespace con un solo clic desde el navegador, donde ya tiene un entorno de desarrollo completo y listo para usar.

4. Sube los cambios y recibe feedback: Usa `git push` para enviar sus cambios al repositorio. GitHub Action se ejecuta automáticamente, y puede ver los resultados en la pestaña de "Actions".

5. Revisa el feedback: Ve los resultados de la autocorrección para saber qué debe corregir, y revisa los comentarios del profesor para un feedback más profundo.
